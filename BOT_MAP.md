# Карта бота — как он работает (по коду, 100%)

Документ строится по построчному чтению `bot/handlers/`, `bot/middlewares/`, `bot/keyboards/`.
Для каждого действия: что видит юзер · какие деньги/лимиты тратятся · где гейт (флаг/раздел/Premium) · где может сломаться (и есть ли рефанд).

---

## 0. Архитектура и порядок обработки

**Точка входа:** `bot/main.py` → собирает `Dispatcher`, регистрирует middleware + роутеры (`setup_routers()`), запускает polling или webhook.

**Middleware-цепочка** (внешние → внутренние), `bot/middlewares/`:
`Throttle → DBSession → UserContext → Ban → Maintenance → ChannelGate`
- **Throttle** — антифлуд (лимит сообщений/окно, настраивается в админке).
- **DBSession (core.py)** — открывает `session` на апдейт (инжектится в хендлеры).
- **UserContext** — грузит/создаёт `User`, инжектит `user` + `_` (Translator его языка).
- **Ban** — забаненных не пускает.
- **Maintenance** — режим обслуживания: не-админам отвечает сообщением из админки, апдейт не идёт дальше. Админы (`settings.admin_ids`) проходят.
- **ChannelGate** — обязательная подписка на канал (если флаг `channel_gate` вкл).

**Порядок роутеров** (`bot/handlers/__init__.py`, первый матч выигрывает):
`start → misc → account → bonus → search → model → settings → roles → support → invite → contests → links → inline → menus → photo → kling → video → music_gen → packs_buy → gift → premium → promo → context → documents → groups → chat`
- `kling` перед `video` (точные `video:kling_*`), `gift` перед `premium` (платежи `gift:`), `groups` перед `chat`, **`chat` — последний (catch-all на текст/фото/голос)**.

## 1. Старт и вход — `start.py`

`/start`:
- Парсит deep-link payload: `ref_<id>` (реферал), `redeem_<code>` (подарок), `promo_<CODE>` (промокод), иначе — токен источника трафика.
- **Новый юзер без языка** → сначала экран выбора языка (`onboarding_language_keyboard`); после выбора (`cb_onboarding_lang`) — приветствие.
- **Вернувшийся** → сразу `_send_welcome`: опциональный брендовый медиа-баннер + локализованный текст приветствия + нижнее reply-меню (`main_menu`).

**Главное меню** (`bot/keyboards/reply.py`): кнопка Mini App (только если `MINIAPP_URL` = https) + 8 кнопок:
`Модель · Изображения | Поиск · Видео | Документы · Музыка | Premium · Аккаунт`.

## 2. Чат с ИИ — `chat.py` (catch-all, последний роутер)

Точки входа (только **приватный** чат; в группах — `groups.py`):
- **`on_text`** — любой текст без `/`.
- **`on_photo`** — фото (с подписью + флаг `photo_edit` → выбор «описать/редактировать»; иначе → vision-описание).
- **`on_voice`** — голосовое (флаг `voice_input`, **только Premium**) → STT → echo «услышал» → как текст.

**Общий пайплайн `_answer_text`:**
1. **Модерация** промпта (`moderate`) → запрещено → «mod.blocked».
2. Если у Premium-юзера активен загруженный документ → чат превращается в Q&A по документу (цена `document_cost`).
3. **Квота `consume_text` (Gate#1)** → превышена → сообщение о лимите + кнопка «Аккаунт». (Free и Premium лимиты — из админки.)
4. Контекст (если `context_enabled`, ~N пар из Redis) + системная роль (если `role_enabled` + `custom_role`).
5. Конфиг чата из админки: `streaming_enabled`, `markdown_enabled`, `memory_pairs`.
6. **Стриминг** (если вкл и не документ): `_stream_reply` — плейсхолдер «…», правит сообщение по мере прихода токенов (троттлинг ≥1.2с/≥30 симв, стоп на 4096). Не вышло — фолбэк на буфер.
7. **Буфер**: `ai_chat` → не ok (провайдер лёг/лимит) → **рефанд квоты** + текст ошибки; ok → сохранить контекст + `_send_reply`.
8. **Реклама** после ответа (`_maybe_ad`) — free видят, Premium нет.

**Кнопки под ответом:** 🔊 Озвучить (Premium, TTS) · 🌐 Перевести (тратит текст-квоту, защита от двойного тапа 15с) · 👍 / 👎 (рейтинг).

**Деньги/лимиты в чате:** текст-квота `consume_text`/`refund_text` (рефанд при сбое провайдера). Документ Q&A — `document_cost`. Редактирование фото (img2img, `nano_banana`) — 1 кредит image-пака, рефанд при сбое. Перевод/озвучка/vision — текст-квота.

**`/report <текст>`** → жалоба (`feedback.record_complaint`).

## 3. Медиа-генерация (на старте разделы ВЫКЛ → «Скоро»; включаются в админке)

Общий денежный паттерн (безопасный, из аудита денег): `packs.try_consume(commit=False)` → создать `GenerationJob(status=pending)` → **один `commit()`** (списание+задача атомарно) → `enqueue_or_refund` (рефанд, если очередь недоступна). Async-задачи делает ARQ-воркер (`workers/`): дёргает провайдера, доставляет результат, **рефанд при сбое**; залипшие подметает cron (`billing_tasks`). Перед списанием — проверка `provider.is_available()` и **модерация** промпта.

### 3.1 Изображения — `photo.py`
- **Промпт-сервисы** (GPT Image 2, Nano Banana, Seedream, Midjourney, FLUX 2): открывают config-подменю (модель/качество/соотношение/seed/кол-во вариантов) в FSM → юзер шлёт промпт → **синхронная** генерация (`_run_photo`): списать → `generate_image` → доставить с кнопками «ещё вариант / апскейл / файл». Бюджет: **недельная квота** для GPT Image 2 / Nano Banana 2, иначе **image-пак**. Непришедшие варианты — рефанд.
- **Инструменты:** **Аватары** (`/ava`, 200⭐ разово, инвойс Stars, async ~15 мин, провайдера пока нет → воркер возвращает Stars), **Face Swap** (2 фото, 1 image-кредит, async), **Upscale** (X2=2/X4=4 кредита, async). Цены — live из админ-конфига `phototools`.
- Гейты: раздел «images» (меню) + флаги `avatar/faceswap/upscale/recraft` (на сервис).

### 3.2 Видео — `video.py` + `kling.py`
- **6 config-сервисов** (Seedance, Veo, Grok, Kling AI, Hailuo, Pika): config → промпт (или фото для image-to-video) → **video-пак** (Midjourney Video — image-пак) → async-задача `process_video_job`.
- **Kling Effects** (74 шаблона, 7 страниц) / **Kling Motion** (13): листаешь каталог → выбираешь шаблон → грузишь фото → 1 video-кредит → async. Каталоги сидятся `scripts/seed_catalogs.py`.
- «Видеоэффекты» — мост в Mini App.

### 3.3 Музыка — `music_gen.py`
- **Suno / Lyria**, полностью платно (нужен music-пак, без бесплатного). Выбор сервиса → проверка баланса → промпт → 1 music-кредит → async `process_music_job` → аудио.

## 4. Платежи и деньги-вход — `premium.py`, `packs_buy.py`, `gift.py`, `promo.py`

**Premium-подписка** (`premium.py`, FSM продукт→срок→шлюз→инвойс):
- Кнопка/`/premium` → выбор продукта (`prem:`) → срок (`premdur:`, при смене тарифа — предупреждение «время стекается под новый тариф») → шлюз (`pay:`).
- **Telegram Stars** — нативно: `answer_invoice` (валюта XTR), payload `sub:<product>:<months>`. **Внешние шлюзы** (СБП/ЮКасса/Stripe) — `create_checkout` → кнопка-ссылка оплаты → активация позже через **вебхук** (`api/routers/webhooks.py`).
- **`pre_checkout`** — ЕДИНСТВЕННЫЙ в проекте (у gift нет): одобряет `sub:/pack:/credits:/gift:/avatar`.
- **`on_successful_payment`** → `_apply_stars_payment`: выдаёт право **идемпотентно по charge_id** (sub→подписка, pack→пак-кредиты, credits→кредиты, avatar→разовая покупка + запрос селфи). Неизвестный payload или сбой активации → **рефанд Stars** (`_refund_stars`, деньги-первыми). Реферальная награда — на первой платной покупке (идемпотентна, avatar исключён).

**Паки генераций** (`packs_buy.py`): пак→кол-во→шлюз→Stars-инвойс (`pack:<pack>:<qty>`). Вход с Gate#2 «пополнить», music-paywall, экрана аккаунта.

**Промокоды** (`promo.py`): `/promo <код>` или deep-link `promo_` → `promos.redeem` (атомарно, race-safe) → кредиты/пак. Статусы: ok / already / invalid.

**Подарки** (`gift.py`): покупка Premium «в подарок» (payload `gift:`), сам успешный платёж ловит gift-роутер (зарегистрирован ПЕРЕД premium); одариваемый активирует через deep-link `redeem_<code>` на `/start`.

## 5. Аккаунт, настройки, реферал, бонус

- **Аккаунт** (`account.py`): `/account` / кнопка → лимит на сегодня (used/limit), статус (free / Premium / Premium×2), выбранная модель, балансы image/video/music, контакт поддержки. `on_contact` — сохраняет телефон + страну, если юзер делится СВОИМ контактом.
- **Модель** (`model.py`): `/model` / кнопка → список моделей из БД-каталога (понятные имена). Premium-модели заблокированы для free (`model.premium_locked`). `set_model` при выборе.
- **Настройки** (`settings.py`): хаб → Модель · Роль · Контекст (память вкл/выкл) · Голос (Premium) · Язык.
  - **Роль** — кастомный system-prompt (≤1000 симв), подставляется в КАЖДЫЙ запрос; `/clear`/`-` сбрасывает.
  - **Голос** (Premium) — выбор TTS-голоса + вкл/выкл + превью.
  - **Язык** — меняет язык, перерисовывает настройки + reply-меню + командную панель Telegram.
- **Бонус** (`bonus.py`): `/bonus` / 🎁 → ежедневный бонус по стрику (`daily_bonus.claim`, идемпотентно/день).
- **Реферал** (`invite.py`): `/invite` /ref → ссылка `?start=ref_<id>` + кол-во приглашённых + награда (live из настроек рефералки).

## 6. Прочие функции

- **Роли/персоны** (`roles.py`): `/roles` → список пресет-персон (из `business_config`, админ-редактируемый) → ставит как `custom_role`. `role:off` сбрасывает.
- **Конкурсы** (`contests.py`): `/contests` → открытые розыгрыши с кнопкой «Участвовать» (идемпотентно). Заголовки админа HTML-эскейпятся.
- **Inline-режим** (`inline.py`): `@bot <вопрос>` в любом чате → **бесплатный** AI-ответ (квоту НЕ тратит), но Redis-rate-limit 20/60с на юзера + прямая проверка бана (middleware для inline не работает). Свой DB-сеанс.
- **Кнопки-ссылки** (`links.py`): `/links` → кастомные URL-кнопки из `business_config` (админ-редактируемые), с трекингом кликов через `/r/{id}` при публичном домене. Только безопасные схемы.
- **Подарки** (`gift.py`): `/gift` → Premium (1 мес) / image-пак (50) → Stars-инвойс (payload `gift:`). `on_gift_payment` ловит **только** `gift:` (узкий фильтр — чтобы не съесть чужие платежи) → создаёт подарок + ссылку `redeem_<code>`. `/redeem <код>` активирует.

## 7. Middleware-цепочка (детали)

1. **Throttle** (внешний, до БД): Redis fixed-window лимит на юзера (лимит/окно — из админки). Флуд дропается с уведомлением, **без запроса в Postgres**.
2. **DBSession**: открывает `session` на апдейт.
3. **UserContext**: `get_or_create_user` → инжектит `user` + `_`(Translator). Мульти-бот: маппит принимающего бота в `bot_id`.
4. **Ban**: забаненный блокируется на ВСЕХ поверхностях (единый источник правды).
5. **Maintenance**: режим обслуживания — не-админам сообщение, дальше не идёт.
6. **ChannelGate (Gate#1)**: при флаге `channel_gate` — free не-админам нужна подписка на канал(ы). Белый список команд (start/premium/account/privacy/help + колбэки gate/premium/pay/close), чтобы можно было подписаться/оплатить. Admin + Premium проходят. `gate:check` (в `misc.py`) перепроверяет подписку и выдаёт удержанную реф-награду за регистрацию.

## 8. Группы и сервисные команды

- **Группы** (`groups.py`): в group/supergroup бот отвечает ТОЛЬКО при обращении (@упоминание или reply на него) И при флаге `groups_enabled`. Снимает упоминание → тот же чат-пайплайн. Мульти-бот-aware, best-effort.
- **Сервисные** (`misc.py`/`support.py`/`context.py`): `/help`, `/privacy` (ссылки на оферту/политику), `/support <текст>` (в инбокс админки, ответ приходит обратно), `/deletecontext` (очистить память), `/report <текст>` (жалоба), отправка контакта → сохранить телефон/страну.

## 9. Деньги и безопасность (сводка)

- **Текст-квота (Gate#1)**: `consume_text` на чат/перевод/vision/voice/часть фото; **рефанд `refund_text`** при сбое провайдера. Лимиты free/Premium — из админки.
- **Паки** (image/video/music): `packs.try_consume` / `refund`. Async-генерация — **атомарный charge+job** (`try_consume(commit=False)` + `GenerationJob` + один commit) → `enqueue_or_refund`; воркер рефандит при сбое; cron подметает залипшие.
- **Stars-платежи**: активация **идемпотентна по charge_id**; неизвестный payload / сбой активации → **рефанд Stars деньги-первыми**. Внешние шлюзы — активация по вебхуку.
- **Inline бесплатен**, но Redis-rate-limit + прямая проверка бана (middleware для inline не работает).
- **Модерация** промптов на всех точках ввода. Админ-тексты HTML-эскейпятся.

## 10. Что нужно для запуска (config, не код)

- **Чат**: добавить AI-аккаунт (OmniRoute/LiteLLM) в админке + `AI_BASE_URL_ALLOWLIST=omniroute,litellm`. Модели сидятся авто. ✅ путь end-to-end проверен.
- **Разделы Видео/Фото/Музыка/Документы/Поиск**: по умолчанию ВЫКЛ (чат-only) → показывают редактируемый текст «Скоро»; включаются тумблером в админке (Цены → Разделы), когда подключены медиа-провайдеры.
- **Mini App кнопка**: только при `MINIAPP_URL`=https; пусто → скрыта.
- **Платёжные шлюзы / медиа-провайдеры / поиск (Perplexity) / STT/TTS / vision** — каждый включается своим ключом/флагом; без него честный «Скоро»/рефанд.
