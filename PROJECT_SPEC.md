## A. Telegram Bot

### 🟢 Зафиксировано ранее
- **Бренд**: **SUPER ИИ БОТ** (ru) / **SUPER AI BOT** (остальные языки). По-разному в локалях — вшито в приветствие каждой локали; единый инфра-`BRAND_NAME` (админка/TOTP/заголовки) = `SUPER AI BOT`. Поддержка: `@lilcipher`.
- **Приветствие**: оставлено как есть (подробное, под тарифы), добавлено только имя бота в первую строку.
- **Старт**: новый юзер → выбор языка → приветствие + нижнее меню.
- **Запуск — ПОЛНЫЙ ПРОДУКТ**: все разделы (Видео/Фото/Музыка/Документы/Поиск) по умолчанию **ВКЛ**. Никакого «Скоро» на старте. Раздел реально генерит, как только в админке (API-ключи) введён ключ провайдера; до этого попытка генерации **безопасно делает refund** (деньги не теряются). «Скоро» — только если админ сам **выключит** раздел (тумблер Цены → Разделы), текст редактируемый.
- **Меню**: нижнее reply-меню — все 8 кнопок всегда видны (Модель/Изображения/Поиск/Видео/Документы/Музыка/Premium/Аккаунт). Раскладка статичная.
- **Чат** идёт через AI-аккаунт (на старте — OmniRoute/LiteLLM, юзер видит только понятные имена моделей, не внутрянку). Проверено end-to-end.
- **Модель**: юзер выбирает из списка понятных имён; Premium-модели заблокированы для free.
- **Оплата**: бесплатно + лимит; Premium снимает лимит + даёт лучшие модели. Premium-подписка продаётся на старте.
- **Паки** (image/video/music): продаются ТОЛЬКО если их раздел включён (иначе кнопка скрыта + «Скоро»). Управляется тумблером раздела.
- **Mini App кнопка** в меню убрана на старте (вернётся при `MINIAPP_URL`).

### 🟢 Связи / интеграции (из кода)
- **AI-текст** → AI-аккаунты в БД (kind=omniroute/litellm/openai/…) через OpenAI-совместимый `base_url`. Авто-сид 9 моделей. Нужен `AI_BASE_URL_ALLOWLIST=omniroute,litellm`.
- **Платежи**: Telegram Stars (нативно) + внешние шлюзы (СБП/ЮКасса/Stripe/CryptoBot/Tribute) через вебхуки API.
- **Медиа** (фото/видео/музыка): провайдеры-шлюзы (Kie/MuAPI/…) — на старте не подключены → разделы выкл.
- **Поиск**: Perplexity (или фолбэк на текст-модель). **STT/TTS/Vision**: свои адаптеры/ключи.

- 🟢 **Документы** — ВКЛ, **только Premium** (работает на чат-модели, провайдер не нужен).
- 🟢 **ПОЛНЫЙ ПРОДУКТ НА СТАРТЕ:** все разделы ВКЛ — чат, видео, фото, музыка, поиск, документы. Код всех разделов **уже есть**; генерация идёт через `media_dispatch` → провайдер (агрегатор-аккаунт Kie/MuAPI **или** прямой ключ). «Реализовать» свелось к: ввести ключ провайдера в админке (страница API-ключи) → раздел сразу работает. Провайдеры по разделам:
  - **Фото**: OpenAI (GPT Image), Seedream, Flux/BFL, Midjourney, + агрегаторы.
  - **Видео**: Kling, MiniMax (Hailuo), Pika, + Veo/Seedance/Grok через агрегатор.
  - **Музыка**: Suno, Lyria.
  - **Поиск**: Perplexity (фолбэк на текст-модель).
  - **Документы**: чат-модель (ключ не нужен).
  - Без ключа раздела → генерация делает **refund** (не молчаливая ошибка).

- 🟢 **Стриминг ответов — ВКЛ** (живая печать, как ChatGPT). Дефолт `streaming_enabled=True`.

- 🟢 **Гейт подписки на канал — ВКЛ для free** (free обязан подписаться; Premium+админы проходят). Админ-управляемо: какие каналы / сменить / добавить-убрать (страница «Функции» → флаг `channel_gate` + список каналов). _Юзер сам добавит свой канал и включит гейт._

- 🟢 **Оплата — все способы**: Telegram Stars (нативно, сразу) + СБП/ЮКасса + Stripe + Crypto (CryptoBot) + Tribute. Каждый активируется ключами в админке (Платежи); добавить/убрать/отключить — там же.
- 🔴 **ПРОБЕЛ к фиксу:** бот показывает ВСЕ шлюзы статически (`GATEWAYS`), а должен — **только настроенные/включённые** (иначе тап по ненастроенному → ошибка). Фикс: `gateway_keyboard`/`pack_gateway_keyboard` фильтруют по «ready»-шлюзам из конфига Платежей (Stars всегда ready). _Реализовать._

- 🟢 **✨ = доп. запросы (единая валюта наград).** Иконка кредитов везде **✨** (не 🪙 — заменено). ✨ — это «запасные запросы»: бот сначала тратит недельный/дневной лимит, а когда он кончился — списывает ✨ (1 ✨ = 1 генерация). Лимит исчерпан И ✨ кончились → только тогда «лимит исчерпан».
  - **Реферал:** за каждого приглашённого друга реферрер получает **N ✨** (одноразово, не сгорают). N — из админки (Рефералы, дефолт 50).
  - **Ежедневный бонус:** ✨ в день по стрик-формуле (база/шаг/кап — из админки, дефолт 5/+1/25).
  - Также ✨ дают: welcome-бонус, first-purchase, cashback, промокоды, покупка за Stars (всё админ-управляемо).
  - Проводка: `quota.consume_text` после базового лимита списывает `user.credits` (✨); `refund_text` возвращает туда же (через `credits_charged` в `QuotaState`). /account показывает ✨-баланс. Free и Premium — оба переливаются в ✨.

- 🟢 **ОНБОРДИНГ АГРЕГАТОРА — карта `ключ бота → upstream slug` (Kie + MuAPI). ЧЕРНОВИК: slug'и сверять с живым каталогом провайдера.**
  - **Как маршрут включается:** диспатч роутит сервис через агрегатор, ТОЛЬКО если есть `AIModel`, у которой `Key` == `job.service` (для фото — `job.model_variant`), с нужной `modality` и `account_kind=kie|muapi`. Нет модели → агрегатор пропускается → direct-заглушка → refund.
  - **Один аккаунт = одна модальность.** Чтобы один ключ обслуживал фото+видео+музыку — завести **3 `AIAccount`** с тем же ключом, разной `modality` (image/video/music). Kie base_url `https://api.kie.ai`, MuAPI `https://api.muapi.ai`.
  - **Kie:** `POST /api/v1/jobs/createTask {model: <slug>, input}` (Bearer). `upstream_model` = `<slug>`.
  - **MuAPI:** `POST /api/v1/{slug}` (x-api-key) — slug идёт **в путь URL**. `upstream_model` = `<slug>`.
